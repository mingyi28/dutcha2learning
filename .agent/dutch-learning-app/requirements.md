# 需求文档

## 引言
本项目旨在开发一个适配手机端的荷兰语学习任务打卡网页。该应用专为初学者设计，采用每日任务制，帮助用户每天学习10个新单词并复习过往内容。应用将提供中、英、荷三语翻译及例句，并具备打卡功能以激励用户坚持学习。

## 需求

### 需求 1：每日新词学习

**用户故事：** 作为一名荷兰语初学者，我希望每天能学习10个新的荷兰语单词，并查看其对应的翻译和例句，以便扩充我的词汇量。此外，我希望在学完一组后能选择复习或继续学习。

#### 验收标准

1. **WHEN** 用户访问“今日学习”页面，**THEN** 系统应当展示10个当日计划学习的新单词。
2. **IF** 展示单词信息，**THEN** 系统必须包含该单词的荷兰语拼写、中文翻译、英文翻译以及荷兰语例句（附带中文翻译）。
3. **WHEN** 用户完成一个单词的学习点击“下一个”，**THEN** 系统应当切换到下一个单词，直到完成今日所有新词。
4. **WHEN** 用户完成今日打卡后，**THEN** 系统应当提供“再次学习”（复习当前组）和“学习下一组”（加载新的一组单词）的选项。
5. **IF** 用户选择“再次学习”，**THEN** 系统应当提供“打乱顺序”和“跳过已掌握单词”的高级选项。
6. **IF** 用户选择“学习下一组”且还有未学单词，**THEN** 系统应当加载新的10个单词并重置进度条。

### 需求 2：复习中心与生词本

**用户故事：** 作为一名荷兰语初学者，我希望拥有一个综合的复习中心，不仅能随机复习，还能针对性地复习生词和查看过往的学习记录。

#### 验收标准

1. **WHEN** 用户进入“复习”模块，**THEN** 系统应当展示“复习中心”首页，包含“生词本”入口和“近一周学习记录”列表。
2. **IF** 用户点击“生词本”，**THEN** 系统应当进入生词复习模式，仅展示被标记为“生词”的单词。
3. **IF** 用户点击历史记录中的某一天，**THEN** 系统应当进入该日期的复习模式，复习当天学习过的单词。
4. **WHEN** 在复习过程中，**THEN** 用户可以随时将当前单词标记为“生词”（加入生词本）或“已掌握”（移出生词本）。
5. **IF** 单词被标记为“已掌握”，**THEN** 它将从生词本中移除，并在后续的智能复习中降低出现频率（如果实现了算法）。

### 需求 3：双模态复习（认读与拼写）

**用户故事：** 作为一名用户，我希望在复习时不仅能看，还能动手拼写，以加深记忆。

#### 验收标准

1. **WHEN** 用户开始一组复习（无论是每日复习还是生词本复习），**THEN** 系统首先进入“认读模式”，展示单词卡片。
2. **WHEN** 用户完成当前组的认读后，**THEN** 系统应当自动（或通过引导按钮）进入“拼写模式”。
3. **IF** 处于拼写模式，**THEN** 系统应当隐藏荷兰语单词，仅展示中文/英文释义，并提供输入框供用户输入荷兰语。
4. **WHEN** 用户输入并提交，**THEN** 系统应当检查拼写是否正确。
   - 如果正确：显示绿色对勾，自动进入下一个。
   - 如果错误：显示红色叉号，允许重试。
5. **IF** 用户遇到困难，**THEN** 系统应当提供“提示”功能，显示正确拼写。

### 需求 4：打卡与进度追踪

**用户故事：** 作为一名用户，我希望在完成每日的学习和复习任务后进行打卡，以便记录我的学习连续性和成就感。

#### 验收标准

1. **WHEN** 用户完成今日所有新词学习任务，**THEN** 系统应当显示“打卡”按钮。
2. **WHEN** 用户点击“打卡”，**THEN** 系统应当记录今日日期到打卡历史中，并更新用户的连续打卡天数。
3. **IF** 用户重新访问页面，**THEN** 系统应当显示当前的连续打卡天数和总学习单词数。
4. **Note**: 初始阶段数据存储将使用浏览器本地存储（LocalStorage），无需后端数据库。

### 需求 5：移动端适配

**用户故事：** 作为一名手机用户，我希望网页界面能够完美适配手机屏幕，以便我随时随地利用碎片时间学习。

#### 验收标准

1. **IF** 用户使用移动设备（屏幕宽度 < 768px）访问，**THEN** 页面布局应当自动调整为单列流式布局或卡片滑动模式。
2. **WHEN** 用户在手机上操作，**THEN** 按钮和交互元素的大小应当适合手指触控。

### 需求 6：打卡记录日历与详情

**用户故事：** 作为一名用户，我希望通过日历形式查看我的打卡记录，并能点击查看某一天的具体学习内容。

#### 验收标准

1. **WHEN** 用户点击底部导航栏的“打卡记录”，**THEN** 系统应当进入打卡记录页面。
2. **IF** 展示打卡记录，**THEN** 系统应当使用日历视图，仅展示当前月份和下个月份的数据。
3. **IF** 某一天已打卡，**THEN** 该日期在日历上应当有明显的标记（如高亮或图标）。
4. **WHEN** 用户点击日历上的“已打卡”日期，**THEN** 系统应当弹出一个模态框，展示该日期所学习的单词列表（包含荷兰语、英语、中文）。
5. **IF** 页面顶部，**THEN** 系统应当展示当前“已掌握”的单词总数统计。

### 需求 7：页面状态保持

**用户故事：** 作为一名用户，我希望在切换底部导航栏（Tab）时，页面能保持之前的操作状态（如今日学习进度），以便我随时切回继续操作而不必重新开始。

#### 验收标准

1. **WHEN** 用户在“今日学习”页面进行到一半或已完成，**AND** 切换到“复习”或“打卡记录”页面，**THEN** 再次切回“今日学习”页面时，应当展示之前的进度或完成状态，而不是重置。
2. **Note**: 使用 Vue 的 `KeepAlive` 机制实现。
